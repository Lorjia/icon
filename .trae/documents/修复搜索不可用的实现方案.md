## 问题原因
- 输入框未订阅 zustand 状态：`Navigation.tsx:51-55` 使用 `useSearchStore.getState()` 读取值，组件不会因状态变更而重渲染，导致输入框的 `value` 不更新，`Enter` 时仍是空字符串，搜索不触发。
- `term` 可能被二次编码：`services/itunes.ts:29-41` 先对 term 自行编码，再交给 `URLSearchParams`，会把 `+` 编码为 `%2B`，与 iTunes API 期望的“空格用 +”不一致，导致查询词异常。

## 修复方案
1. 正确使用 zustand 订阅搜索状态（Navigation）：
   - 在组件顶部使用选择器订阅：`const term = useSearchStore(s => s.term); const setTerm = useSearchStore(s => s.setTerm); const search = useSearchStore(s => s.search)`。
   - 输入框改为：`value={term}`，`onChange={e => setTerm(e.target.value)}`，`onKeyDown={e => e.key==='Enter' && search()}`。
   - 移动端输入同样调整，保证回车触发搜索并关闭菜单。
2. 避免二次编码（services/itunes.ts）：
   - 去掉 `encodeTerm`，直接 `params.set('term', term.trim())`，由 `URLSearchParams` 统一编码；它会正确处理空格与非 ASCII 字符。
   - 保留 `country/media/entity/limit/lang` 设置不变。
3. 可视化错误与调试：
   - 若搜索失败，`useSearchStore` 的 `error` 已在 `ResultsGrid` 显示；补充在失败时打印 `url` 和 `status`（仅开发环境）以便定位。
4. 快速验证用例：
   - 关键词：`微信`（CN），`WeChat`（US），`LINE`（JP/KR），平台切到 iPhone/iPad/Mac 均应返回结果。

## 变更文件
- `src/components/Navigation.tsx`：改为使用 zustand 订阅的 `term/setTerm/search`。
- `src/services/itunes.ts`：移除 `encodeTerm` 并直接设置 `term` 到 `URLSearchParams`。
- （可选）`src/components/Navigation.tsx` 移动端同样修复；`useSearchStore.getState()` 全部替换为订阅选择器。

## 验收标准
- 输入可正常变更，回车或点击“搜索”均能发起请求。
- CN/US/JP/KR 四个区域均可返回结果并展示。
- 错误时页面显示错误信息；成功时结果网格出现。
